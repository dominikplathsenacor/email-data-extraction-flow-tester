# Documentation

This folder contains implementation-facing documentation for `simple-e2e-tester`.

## Read in this order

1. `architecture.md`: domain-first design and runtime flow.
2. `configuration.md`: complete test configuration schema and examples.
3. `workflows.md`: CLI usage, workbook contracts, matching/validation semantics.
4. `spec-alignment.md`: what is implemented, partial, or intentionally divergent from desired behavior.
5. `testing-and-quality.md`: TDD-oriented test structure and quality commands.

## Source-of-truth documents

- Architecture and domain boundaries: `architecture.md`
- Configuration reference: `configuration.md`
- Workflow and workbook semantics: `workflows.md`
- Testing and quality gates: `testing-and-quality.md`
